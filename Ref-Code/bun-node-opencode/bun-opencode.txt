❯ 我嘗試進入opencode                                                            
container無法找到並執行bun，是因為三個image整合但程式之間是不能通用的嗎?        

● 是的，這是 Docker 的核心概念 — 每個 container 是獨立隔離的環境。

  為什麼不能跨 container 執行程式

  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
  │   opencode      │  │   bun_app       │  │   nginx_proxy   │
  │                 │  │                 │  │                 │
  │  /usr/bin/...   │  │  /usr/local/bin │  │  /usr/sbin/...  │
  │  (opencode 的   │  │  /bun  ← 只在  │  │  nginx ← 只在  │
  │   二進位檔)     │  │  這裡有         │  │  這裡有         │
  └─────────────────┘  └─────────────────┘  └─────────────────┘
           ↑
     在這裡找不到 bun

  每個 container 有獨立的：
  - 檔案系統（包含所有可執行程式）
  - 環境變數
  - 程序空間

  三個 image 整合的意思是它們可以透過網路互相溝通，而不是共享程式。

  如果你需要在 opencode 裡使用 bun，有幾個選擇：

  選項 1：用 network 讓 opencode 呼叫 bun service（正確做法）

  opencode 透過 HTTP 請求呼叫 bun service：
  opencode → http://bun_app:3000/api/...

  選項 2：建立一個同時有兩者的自訂 image

  FROM ghcr.io/anomalyco/opencode:latest
  COPY --from=oven/bun:latest /usr/local/bin/bun /usr/local/bin/bun

  選項 3：用 volumes 共享程式碼，但各自執行

  opencode 和 bun 掛載同一個 volume，opencode 產生的程式碼由 bun container
  執行。

  ---
  你實際的使用情境是什麼？opencode 需要「執行」bun，還是只需要讀取 bun
  產生的結果？這樣比較好建議正確架構。

● 結果如下：

  Dockerfile.opencode — multi-stage build，把 bun 複製進 opencode：
  Stage 1: oven/bun  →  取出 /usr/local/bin/bun
  Stage 2: opencode  →  主體，注入 bun 執行檔

  compose.yml — opencode 改為 build 自訂 image，移除獨立的 bun
  service（已不需要）。

  啟動方式：
  # 第一次需要 build
  docker compose build opencode

  # 之後正常啟動
  docker compose up -d

  # 進入 container 安裝 opencode plugin
  docker exec -it opencode sh
  bun install <plugin>

  ▎ 原理： multi-stage build 只是借用 oven/bun image 的執行檔，最終產出的 image 
  仍以 opencode 為基礎，bun 二進位檔被複製進去，兩者共存於同一個 container。
